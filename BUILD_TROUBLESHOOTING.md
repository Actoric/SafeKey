# Решение проблем сборки

## Проблема с символическими ссылками при сборке

Если при сборке вы видите ошибку:
```
ERROR: Cannot create symbolic link : Клиент не имеет необходимых привилегий.
```

Это происходит потому, что electron-builder пытается распаковать инструменты для подписи кода, которые содержат символические ссылки.

### Решение 1: Запуск от имени администратора (Рекомендуется)

1. Закройте текущий терминал/PowerShell
2. Запустите PowerShell **от имени администратора**:
   - Нажмите Win+X
   - Выберите "Windows PowerShell (Администратор)" или "Терминал (Администратор)"
3. Перейдите в папку проекта:
   ```powershell
   cd E:\Progect\SafeKey
   ```
4. Запустите сборку:
   ```powershell
   npm run build:win
   ```

### Решение 2: Использование распакованной версии

Если у вас уже есть папка `release\win-unpacked`, вы можете использовать приложение напрямую:

1. Перейдите в `release\win-unpacked\`
2. Запустите `SafeKey.exe`

Это полнофункциональная версия приложения, просто без установщика.

### Решение 3: Создание portable версии

Вы можете скопировать папку `release\win-unpacked` в любое место и использовать как portable версию.

## Альтернативная сборка без установщика

Если проблемы с установщиком продолжаются, вы можете использовать только распакованную версию:

```powershell
# Сначала соберите код
npm run build

# Затем соберите только распакованную версию
npx electron-builder --win --dir
```

Это создаст только папку `release\win-unpacked` без установщика.

## Проверка готовности к сборке

Убедитесь, что:
1. ✅ Все зависимости установлены: `npm install`
2. ✅ Код собран: `npm run build`
3. ✅ Папка `dist/` существует и содержит скомпилированные файлы

Если всё выполнено, основная часть приложения готова. Проблема только с финальным упаковыванием в установщик.
