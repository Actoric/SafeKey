# Инструкция по сборке SafeKey

## Текущая ситуация

Приложение успешно собирается, но при создании установщика возникает ошибка с символическими ссылками (требуются права администратора).

## ✅ Решение: Использовать распакованную версию

После выполнения `npm run build:win` приложение уже готово в папке:
```
release\win-unpacked\SafeKey.exe
```

**Это полнофункциональная версия!** Просто запустите `SafeKey.exe`.

### Варианты использования:

1. **Прямой запуск:**
   - Откройте `release\win-unpacked\`
   - Запустите `SafeKey.exe`

2. **Portable версия:**
   - Скопируйте всю папку `win-unpacked` в любое место (например, на USB)
   - Создайте ярлык на `SafeKey.exe` на рабочем столе

## Создание только распакованной версии (без установщика)

Если хотите собрать только приложение без установщика:

```powershell
npm run build:win:dir
```

Это создаст только папку `release\win-unpacked` без попытки создать установщик.

## Создание установщика (требуются права администратора)

Если вам действительно нужен установщик `.exe`:

1. **Запустите PowerShell от имени администратора:**
   - Нажмите `Win + X`
   - Выберите "Терминал (Администратор)" или "Windows PowerShell (Администратор)"

2. **Выполните команду:**
   ```powershell
   cd E:\Progect\SafeKey
   npm run build:win
   ```

Это должно успешно создать установщик `SafeKey-1.0.0-x64.exe` в папке `release\`.

## Примечание

Распакованная версия (`win-unpacked`) работает точно так же, как установленная версия. Единственное отличие - это отсутствие записи в реестре Windows и программ удаления через "Панель управления".

Для большинства пользователей распакованная версия даже удобнее - не требует установки!
